MITM Resource Project

How to Suggest Changes

If you want to suggest an additional resource or point out an error, please use one of the options below.

Back to Resource List View GitHub Repository

Suggestions are reviewed before they are added. Email is the simplest option if you are not familiar with GitHub.

Contact Options

Two ways to suggest a change

Option 1

Email Kunal

Send an email to kunal@wustl.edu with the subject line MiTM resource suggestion.

Please include the resource name, a link if available, the age group, and a short description. If you are reporting an error, include the page or resource name and what should be corrected.

Option 2

Open a Pull Request on GitHub

If you are comfortable with GitHub, you can suggest a direct edit by opening a pull request in the MiTM-resources repository .

Most resource updates will go in resources.js. Pull requests are welcome, but all suggested additions are reviewed before they are accepted.

What Helps

Information that makes review easier

  • The resource name
  • A working official link, if one exists
  • The age group it fits best
  • A one or two sentence description
  • Whether it belongs under camps, competitions, schools, books, videos, or another category
